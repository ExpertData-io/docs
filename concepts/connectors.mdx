## What are Connectors?

**Connectors** are secure connections to your data sources. They store encrypted credentials and connection parameters, allowing you to connect once and query repeatedly.

<Note>All credentials are encrypted using AES-256 before storage.</Note>

## Creating a Connector

<Steps>
  <Step title="Navigate to Connectors">
    Go to **Dashboard** â†’ **Connectors**
  </Step>
  <Step title="Click New Connector">Click the **+ New Connector** button</Step>
  <Step title="Select Type">
    Choose your data source type (e.g., PostgreSQL)
  </Step>
  <Step title="Enter Details">
    Fill in the connection parameters: - **Name**: A friendly name for the
    connector - **Host**: Database server address - **Port**: Connection port -
    **Database**: Database name - **Username**: Auth username - **Password**:
    Auth password (encrypted)
  </Step>
  <Step title="Test Connection">
    Click **Test Connection** to verify connectivity
  </Step>
  <Step title="Save">Click **Save** to create the connector</Step>
</Steps>

## Connector Configuration

### SQL Databases

```json
{
  "connector_name": "Production DB",
  "connector_type": "sql_connector",
  "connector_params": {
    "engine": "postgresql",
    "server": "db.example.com",
    "port": 5432,
    "database": "analytics",
    "username": "user",
    "password": "***encrypted***"
  }
}
```

### REST APIs

```json
{
  "connector_name": "Weather API",
  "connector_type": "api_connector",
  "connector_params": {
    "base_url": "https://api.weather.com/v1",
    "auth_type": "api_key",
    "api_key": "***encrypted***",
    "headers": {
      "Content-Type": "application/json"
    }
  }
}
```
