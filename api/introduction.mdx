## Base URL

All API requests should be made to:

```
https://api.expertengine.io/v1
```

For local development:

```
http://localhost:8030
```

## Authentication

ExpertEngine uses **cookie-based JWT authentication**. See [Authentication](/api/authentication) for details.

## Request Format

- All requests must include the `Content-Type: application/json` header
- State-changing requests (POST, PUT, DELETE) require a `X-CSRF-Token` header
- Request bodies should be JSON-encoded

```bash
curl -X POST https://api.expertengine.io/v1/tasks \
  -H "Content-Type: application/json" \
  -H "X-CSRF-Token: your-csrf-token" \
  -b "access_token=your-jwt-token" \
  -d '{"name": "My Task", "type": "sql"}'
```

## Response Format

All responses are JSON with a consistent structure:

### Success Response

```json
{
  "uuid": "abc123",
  "name": "My Task",
  "created_at": "2024-01-15T10:30:00Z",
  "updated_at": "2024-01-15T10:30:00Z"
}
```

### Error Response

```json
{
  "detail": "Task not found",
  "status_code": 404
}
```

## Status Codes

| Code  | Description                    |
| ----- | ------------------------------ |
| `200` | Success                        |
| `201` | Created                        |
| `204` | No Content (successful delete) |
| `400` | Bad Request (validation error) |
| `401` | Unauthorized                   |
| `403` | Forbidden                      |
| `404` | Not Found                      |
| `422` | Unprocessable Entity           |
| `429` | Rate Limited                   |
| `500` | Server Error                   |

## Rate Limiting

API requests are rate limited:

| Tier          | Limit        |
| ------------- | ------------ |
| Anonymous     | 10 req/min   |
| Authenticated | 100 req/min  |
| Premium       | 1000 req/min |

Rate limit headers are included in responses:

```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1704067200
```

## Pagination

List endpoints support pagination:

```
GET /tasks?page=1&page_size=20
```

Response includes pagination metadata:

```json
{
  "items": [...],
  "total": 100,
  "page": 1,
  "page_size": 20,
  "pages": 5
}
```
